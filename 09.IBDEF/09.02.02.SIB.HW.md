# 09.02.02. Active Directory (часть 2)
### Задание 1
> Проведите самостоятельный аудит установленного на прошлом занятии DC, опираясь на лекцию.
> 
> Проведете проверку системы при помощи [`DCDiag`](https://learn.microsoft.com/ru-ru/windows-server/administration/windows-commands/dcdiag).
> 
> *Дайте ответ в свободной форме.*

**Ответ:**

[Лог проверки DCDiag](_att/090202/090202_dcdiag.md)

Выводы:

Сервер функционирует, но требует настройки по следующим направлениям:

1. Репликация SYSVOL (DFSREvent) - может приводить к проблемам с применением групповых политик (GPO) на клиентских компьютерах;

2. Настройка DNS для корректного разрешения имен - указывает на отсутствие или некорректность SRV-записей, недоступность DNS-сервера;

3. Настройка времени на PDC - сервер не настроен на внешний источник времени (NTP);

4. Усиление безопасности LDAP - сервер принимает незашифрованные LDAP-привязки и не требует подписи.

### Задание 2*
> Установите утилиту `AD ACL Scanner` и проведите аудит DC.
> 
> *Дайте ответ в свободной форме.*

**Ответ:**

Получены отчеты [DACL (права доступа)](_att/090202/090202_DML-dml_DACL.pdf) и [SACL (права аудита)](_att/090202/090202_DML-dml_SACL.pdf) для корня домена.

**Выводы по аудиту AD:**

Критические проблемы:

1. Группа "Все" имеет права на чтение всех свойств корня домена - это уязвимость, позволяющая неаутентифицированным пользователям получать информацию о домене.
2. Чрезмерные права у "Пред-Windows 2000 доступ" - группа имеет доступ к конфиденциальным данным (политики паролей, членство в группах).

Проблемы конфигурации:

3. Дублирование административных прав - несколько групп имеют одинаковые полные права, что усложняет управление и аудит.
4. Недостаточный аудит - настроен только аудит успешных операций, отсутствует аудит неудачных попыток для критических действий.
